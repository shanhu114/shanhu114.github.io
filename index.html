Boot —— 引导加载程序文件
Srv—— 服务的数据
home —— 家目录 用户文件目录 存放用户数据 lib —— 存放函数库 run —— 这次开机包含的程序，关机会被删除，下次开机会重新生成
var —— 记录变量文件，存放系统日志 bin —— 放入的工具
etc —— 存放及系统配置文件及软件 dev—— 外接设备
home —— 家目录 用户文件目录 存放用户数据
tmp —— 临时目录（不能放重要文件，有可能被清除掉）
opt —— 存放可选的程序（一般存放测试的文件夹）
media —— 挂载USB DVD 等等驱动
usr —— 不适合放在bin目录下面的存放的文件夹
mnt —— 挂载临时挂载设备目录—比如光驱
proc —— 信息存放的文件包含CPU 内存
sbin —— 超级bin所使用的命令 —-超级权限的用户–普通用户不能执行
boot——系统的内核以及程序文件—–引导加载程序文件
sys —— 存放系统内核相关文件

cd /: 根目录，一般根目录下只存放目录，不要存放文件，/etc、/bin、/dev、/lib、/sbin应该和根目录放置在一个分区中

cd /bin:/usr/bin: 可执行二进制文件的目录，如常用的命令ls、tar、mv、cat等。

cd /boot： 放置linux系统启动时用到的一些文件。/boot/vmlinuz为linux的内核文件，以及/boot/gurb。建议单独分区，分区大小100M即可

cd /dev： 存放linux系统下的设备文件，访问该目录下某个文件，相当于访问某个设备，常用的是挂载光驱mount /dev/cdrom /mnt。

cd /etc： 系统配置文件存放的目录，不建议在此目录下存放可执行文件，重要的配置文件有/etc/inittab、/etc/fstab、/etc/init.d、/etc/X11、/etc/sysconfig、/etc/xinetd.d修改配置文件之前记得备份。注：/etc/X11存放与x windows有关的设置。

cd /home： 系统默认的用户家目录，新增用户账号时，用户的家目录都存放在此目录下，表示当前用户的家目录，test表示用户test的家目录。建议单独分区，并设置较大的磁盘空间，方便用户存放数据

cd /lib:/usr/lib:/usr/local/lib： 系统使用的函数库的目录，程序在执行过程中，需要调用一些额外的参数时需要函数库的协助，比较重要的目录为/lib/modules。

cd /lost+fount： 系统异常产生错误时，会将一些遗失的片段放置于此目录下，通常这个目录会自动出现在装置目录下。如加载硬盘于/disk 中，此目录下就会自动产生目录/disk/lost+found

cd /mnt:/media： 光盘默认挂载点，通常光盘挂载于/mnt/cdrom下，也不一定，可以选择任意位置进行挂载。

cd /opt： 给主机额外安装软件所摆放的目录。如：FC4使用的Fedora 社群开发软件，如果想要自行安装新的KDE 桌面软件，可以将该软件安装在该目录下。以前的 Linux 系统中，习惯放置在 /usr/local 目录下

cd /proc： 此目录的数据都在内存中，如系统核心，外部设备，网络状态，由于数据都存放于内存中，所以不占用磁盘空间，比较重要的目录有/proc/cpuinfo、/proc/interrupts、/proc/dma、/proc/ioports、/proc/net/*等

cd /root： 系统管理员root的家目录，系统第一个启动的分区为/，所以最好将/root和/放置在一个分区下。

cd /sbin:/usr/sbin:/usr/local/sbin： 放置系统管理员使用的可执行命令，如fdisk、shutdown、mount等。与/bin不同的是，这几个目录是给系统管理员root使用的命令，一般用户只能”查看”而不能设置和使用。

cd /tmp： 一般用户或正在执行的程序临时存放文件的目录,任何人都可以访问,重要数据不可放置在此目录下

cd /srv： 服务启动之后需要访问的数据目录，如www服务需要访问的网页数据存放在/srv/www内

cd /usr： 应用程序存放目录，/usr/bin 存放应用程序， /usr/share 存放共享数据，/usr/lib 存放不能直接运行的，却是许多程序运行所必需的一些函数库文件。/usr/local:存放软件升级包。/usr/share/doc: 系统说明文件存放目录。/usr/share/man: 程序说明文件存放目录，使用 man ls时会查询/usr/share/man/man1/ls.1.gz的内容建议单独分区，设置较大的磁盘空间

cd /var： 放置系统执行过程中经常变化的文件，如随时更改的日志文件 /var/log，/var/log/message： 所有的登录文件存放目录，/var/spool/mail： 邮件存放的目录， /var/run: 程序或服务启动后，其PID存放在该目录下。建议单独分区，设置较大的磁盘空间

添加80端口通过的规则
firewall-cmd –zone=public –add-port=80/tcp –permanent
–zone drop：默认丢弃所有包
block：拒绝所有外部连接，允许内部发起的连接
public：指定外部连接可以进入
–add-port 指定添加端口 80/tcp 53/udp
–add-service 指定常见服务 http https mysql mssql ssh
–permanent 永久生效
删除80端口通过的规则
firewall-cmd –zone=public –remove-port=80/tcp –permanent
–remove-port 指定删除端口
–remove-service 指定删除服务
防火墙更新规则
firewall-cmd –reload
防火墙显示所有规则
firewall-cmd –list-all
关闭防火墙
systemctl stop firewalld
查看防火墙状态
systemctl status firewalld
启动防火墙
systemctl start firewalld

LINUX常用命令

Generate static files

OS 操作系统 （——没有操作系统叫做裸机 ）
Linux （内核）开源 可以自由传播
Linux主要特征
可移动兼容接口 posix 1.0 多用户 多任务 支持多种平台
linux 版本
centOS （多用于数据库） Redhat （多用于开发） ubuntu kali （针对做网络安全和数据取证和攻防演练） suse
Linux系统组成
Linux 内核 ： 储存管理 进程管理 驱动
shell ： 内核和用户交互接口 命令解释器 编程语言
文件系统： 文件存在
应用程序 包括文本编辑器 编程语言 办公自动化软件 数据库等

##文件系统
MBR 4个主分区，最高15个分区 只支持2TB，不支持扩容
GPT 无分区限制 ，支持2TB以上，支持数据扩容
查看分区情况 fdisk -l 查看现有磁盘分区列表
df-a 全部显示
df -l 只显示本地磁盘
df -h 查看分区情况，是以1024为进制显示磁盘容量
df-H 是以1000进制显示磁盘容量
df-T 是显示磁盘分区类型
df-t 指定类型文件分区
df-x 不显示指定类型文件分区
du -b 以byte显示 du -k 以KB显示 du -m 以MB显示
du -h （1024） -H (1000)
du -s 统计指定的文件夹

##查看CPU信息
/proc/cpuinfo cpu信息
/proc/meninfo 内存信息
uname -a 查看linux全部信息
uname -v 内核的编译版本号
uname -r 内核的信息release （发行号)
uname -n 主机名
uname -r |grep -q ^ 2.6 ; echo $?
Ismod 列出内核的所有模块
tcpdump - i eth0 抓本机网卡包 -h 后跟ip
zenamp — 启动图形化nmap

##ssh服务安装配置
1.检查是否安装ssh服务
rpm -qa |grep ssh
2.没有执行这条命令安装
yum install openssh-server
3.修改配置文件
vi /etc/ssh/sshd_config

#PORT 22 去掉#号

#PermitRootLogin yes | no yes允许root ssh登录，no不允许
4.启动ssh服务
systemctl start sshd.service
5.设置开机自启（可选）
systemctl enable sshd.service
